### #5. 코드 파일 구조

- **결정일**: 2026-04-26
- **결정**: 계산·시나리오 관련 코드는 인라인 `<script>`가 아닌 별도 .js 파일로 분리. ES6 module은 사용하지 않고 비-모듈 `<script src="...">` 다중 로드 방식으로 한다.
- **이유**:
  - v0.3까지 가면 코드량이 600~800줄로 폭증해 단일 HTML로는 관리 한계
  - PRD 11절 KPI "규칙 테스트 통과율 95% 이상" 충족을 위해 함수가 import 가능해야 함
  - index.html과 result.html이 같은 함수를 공유하려면 별도 파일이 필수
  - 시스템 프롬프트의 "화면과 계산 로직 분리 원칙"과 일치
  - ES6 module 대신 비-모듈 방식을 택한 이유: 로컬 file:// 환경과 GitHub Pages 모두 추가 도구 없이 동작
- **권장 디렉토리 구조**: 의사결정 #7 참조
- **PRD와의 관계**: PRD 8-8 "계산엔진: JavaScript(클라이언트 사이드)"는 "어디서 돌릴 것인가" 결정. 본 의사결정은 "어떻게 모듈화할 것인가" 결정으로 PRD와 충돌하지 않음.
- **영향 범위**: 모든 코드 파일 구조, Code 작업지시서
- **파일명 명명 규칙** (2026-04-27 v3 추가):
  - 모든 .js 파일명은 **언더바(snake_case)** 사용
  - 예: `tax_rules.js`, `tax_engine.js`, `scenario_engine.js`, `input_collector.js`, `result_renderer.js`, `explanation_engine.js`
  - 테스트 파일도 동일: `tests/test_cases.js`
  - 이유: Python 관행 + GitHub URL 가독성 + 일관성

### #8. v0.1 계산 엔진 명세서 검증 완료 + 버전별 폴더 구조

- **결정일**: 2026-04-28
- **결정**:
  1. **v0.1.1 명세서를 v0.1 코드 구현의 기준으로 확정**한다. 작업 창 #1-1(`tax_rules.js` + `calculateSingleTransfer()` 의사코드 + 작업지시서 01) 진입 가능.
  2. **버전별 명세서·테스트 케이스는 `docs/v{버전}/` 폴더 하위**에 둔다. 공통 산출물은 `docs/` 루트에 둔다.

- **검증 절차** (v0.1):
  1. Claude가 명세서 v0.1.1 산식대로 5개 케이스 산출 (TC-001~005)
  2. 검증팀(설하영·이준기·김태환·김두섭)이 명세서를 보지 않고 독립 손계산 (구글시트)
  3. 국세청 홈택스 양도소득세 모의계산기로 동일 입력 모의계산
  4. 3자 결과 비교 → 5건 모두 일치

- **확정된 v0.1.1 산식**:
  - 13단계 파이프라인 (양도차익 → 비과세 → 고가주택 → 장특공 → 양도소득금액 → 기본공제 → 과세표준 → 보유기간 분기 → 적용 세율 → 산출세액 → 지방소득세 → 총 납부세액 → 세후 매각금액)
  - 기본세율표 8단계 누진 (소득세법 제55조 ①, 2026 시행)
  - 단기세율 70% (1년 미만) / 60% (1~2년 미만) (제104조 ① 제2호·제3호)
  - 양도소득 기본공제 250만원 (제103조)
  - 보유기간 동월동일 비교 방식
  - 절사 정책: 산출세액·지방소득세 `Math.floor` (원 미만 절사)
  - 양도차손 시 `taxBase = max(0, ...)` 처리
  - 비조정대상지역 가정, 단일 주택 일반과세

- **검증 후 v0.1에서 보류된 사항** (v0.2·v0.3로 이연):
  - 단기세율 + 기본세율 비교산출세액 규정 (제104조 ① 후단) → v0.3 다주택 중과 + 단기 동시 케이스에서 결정
  - 세후 매각금액 정의 (`salePrice − totalTax`) → v0.2 `transferExpense` 분리 입력 시 재정의 검토
  - 원 미만 절사 단위 (1원 vs 10원) → v0.2 신고서식 관행 확인 후 결정

- **회귀 검증 원칙**: TC-001~005는 v0.2·v0.3 코드 진행 시에도 동일 결과를 유지해야 함. 깨지면 회귀 버그로 간주.
